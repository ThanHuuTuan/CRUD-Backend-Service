# CRUD-Backend-Service
CRUD backend service which consists of a worker (consumer) and the following APIs.  Get: Search in cache first and fallback to the db in case of a cache miss. Post Put Delete Submit: Submit an async job. Return a job_id. Results: Get the results of a previously submitted job  Points to note: Each item in the cache should have some TTL set. Make sure that the cache is always in sync with the db (insert / update / delete operations) Create / Generate the swagger api documentation Make sure you have proper logging in all relevant areas Workers should retry a job in case of any exception (upto N max retries) Make sure that you have proper test scripts to test each of the above APIs.  Tech Stack Python3 + Flask MongoDB / PSQL (pick any one) Celery (job scheduling) RabbitMQ (message broker) Redis (cache)
